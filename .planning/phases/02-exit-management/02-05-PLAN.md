  <action>
Start the bot in paper trading mode and observe the following. For each criterion, the exact verification method is provided.

**Criterion 1: After a BUY executes, hard stop and take-profit are stored in PostgreSQL.**

After the bot places a BUY, run:
```sql
SELECT symbol, direction, entry_price, stop_price, target_price, atr_at_entry
FROM position_registry;
```
Expected: One row per open long position. `stop_price = entry_price - (2 * atr_at_entry)` within rounding tolerance. `target_price = entry_price + (4 * atr_at_entry)`. No NULL values in stop_price or target_price columns.

**Criterion 2: After 1x ATR gain, trailing stop activates and moves up.**

Watch the log for: `[ExitManager] Trailing stop activated for {symbol}: trail={X}`. Then run:
```sql
SELECT symbol, trailing_high, trailing_stop FROM position_registry WHERE symbol = '{symbol}';
```
Expected: `trailing_stop IS NOT NULL` after the log message. `trailing_stop = trailing_high - (1.5 * atr_at_entry)` within rounding.

**Criterion 3: Daily P&L below -2% NAV halts new entries.**

Watch the log for: `[ExitManager] Circuit breaker ACTIVATED`. Then verify subsequent bars show `Circuit breaker active — suppressing new entry signal` for all symbols in that trading session. Existing positions are NOT closed — only new entries are suppressed.

**Criterion 4: Closed positions have structured exit records.**

After any position closes (stop loss, trailing stop, or take profit), run:
```sql
SELECT symbol, action, exit_price, exit_reason, entry_reason, regime_at_decision, decision_time
FROM trade_log
WHERE exit_reason IS NOT NULL
ORDER BY decision_time DESC
LIMIT 5;
```
Expected: `exit_reason` is one of `'HARD_STOP_LOSS'`, `'TRAILING_STOP'`, `'TAKE_PROFIT'`, `'SIGNAL'` — never NULL. `entry_reason` is present for entry records.

**Criterion 5: On bot restart, positions reconciled from IBKR.**

Stop the bot while holding an open position. Restart. Watch the log for:
```
Reconciling position exit registry with IBKR...
[ExitManager] Loaded N positions from DB registry
[ExitManager] Reconciliation complete: N positions tracked
Position reconciliation complete: N positions active
```
Expected: No `BUY` log for a symbol already visible in IBKR positions. The position continues to be monitored with its original stop and target intact.

If any criterion fails, identify the root cause, fix the code, and re-run 02-04 verification before re-checking here.
