# Plan 02-03: Replace Hardcoded Stops in hourly_portfolio_scan() with ExitManager

## Objective

`hourly_portfolio_scan()` in `Scripts/main.py` contains hardcoded stop/take-profit logic at lines 205-210 (`position_return < -0.15` and `position_return > 0.25`). This creates a second competing exit system that bypasses ExitManager, leaves `position_registry` out of sync, and logs exits without `exit_reason`. Replace those hardcoded blocks with `exit_manager.check_exits()` calls. Also write exit events to `TradeLog` from the hourly scan for complete OBS-01 coverage.

This plan can run in parallel with 02-02 (both depend only on 02-01, not on each other).

- @.planning/phases/02-exit-management/02-RESEARCH.md — Pitfall 4 (competing exit systems) and Pitfall 5 (circuit breaker ordering)
